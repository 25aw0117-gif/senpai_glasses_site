## 1. 概要
- **プロジェクト形態**: 前期課題（講師を依頼主とした制作）
- **担当ロール**: 企画、UI/UX設計、フロントエンド実装
- **制作期間**: 2024.05 ~ 2024.08（約3か月）

### サイト制作の目的
1. **情報の透明化**: 在校生の視点から見た「リアルな学生生活」を発信し、入学希望者の理解を深める。
2. **ミスマッチの解消**: 入学後のギャップを減らすことで、学生の意欲維持および退学率の低減に繋げる。
3. **志願者数の向上**: 信頼できる情報提供を通じて、学校の魅力を伝え、入学検討者を増やす。

### ターゲット (Target)
- 日本語能力試験N2以上を保有している外国人留学生。
- Webに興味はあるが、自分の希望職種は決めていな。

### コンセプト (Concept)
**「先輩のメガネ（Senpai's Glasses）」**
- **課題（視界の不透明さ）**: 日本での生活に不安や「見えづらさ」を感じている留学生（ターゲット）。
- **解決策（メガネを貸す）**: 先輩のリアルな体験談という「レンズ」を通すことで、学校生活を鮮明にイメージさせる。
- **成果（鮮明な視界）**: 正確な情報の習得と深い理解を促し、将来への視界をクリアにする。

## 4. 解決した問題 (Problem Solving)
## 外部ライブラリの制約を克服し、デザイン案を忠実に再現するためのSwiperカスタマイズにおける技術的課題と解決プロセス

### 1. スライド幅の制御：自動計算からコンテンツ主体への転換
- **課題**: デフォルトの `width: 100%` 強制適用により、カードが1枚ずつしか表示されない。
- **分析**: ライブラリの移動距離計算ロジックがインラインスタイルを優先していることを確認。
- **根拠と解決**: 計算値より「コンテンツ自体のサイズ」を優先すべきと判断。JSで `slidesPerView: 'auto'` を設定し、CSSで `width: auto !important` を適用することで、柔軟なカード配置を実現しました。

### 2. 視覚的重複の解消：デフォルト資産の構造的無効化
- **課題**: カスタム背景画像とデフォルトの青い矢印アイコン（`::after`）が重なり、デザインの精度が低下。
- **分析**: ライブラリ内蔵のCSSが疑似要素にフォントアセットを強制的に割り当てている。
- **根拠と解決**: 単なる上書きではなく、不要なノードを構造的に除去すべきと判断。`content: none` および `display: none` を指定することで、不要な資産を完全に排除しました。

### 3. レイアウトシステムの統合：絶対配置から流動配置への再構築
- **課題**: ボタンが画面外へ飛ぶ、またはボタン同士が重なり合い、意図した配置（見出しの隣）にならない。
- **分析**: `position: absolute` の特性上、親要素のFlexboxレイアウトを無視して浮いていることが原因。
- **根拠と解決**: ボタンを「独立した浮遊要素」ではなく「ヘッダーの構成要素」として扱うべきと判断。`position: static !important` を適用して絶対配置を解除し、ドキュメントフローに戻すことで、見出しと連動する整然としたレイアウトを構築しました。
